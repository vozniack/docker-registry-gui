Oops... this repository have been deleted

What does it mean?

It looks like the tag or manifest doesn't exists, but registry still contains related directories. It's not possible to delete repository just by Docker Registry API, so if you want to remove repository fully you need to use garbage collector and manually delete directories on registry machine.
Check more - Garbage Collection

What to do?

To remove a repository completely you must execute the following commands:
bin/registry garbage-collect [--dry-run] /path/to/config.yml
rm -r /var/lib/registry/docker/registry/v2/repositories/repository/tag
Check more - Registry configuration
